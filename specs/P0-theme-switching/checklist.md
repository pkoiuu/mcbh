# P0 Checklist: 启动器主题切换（深色/浅色）

## 功能验证

### 主题切换
- [ ] 设置 > 外观页显示"主题模式"行，右侧有 toggle 开关
- [ ] Toggle 开关样式与"启动后自动全屏"等其他开关一致
- [ ] 点击 toggle 即时切换深色/浅色主题
- [ ] Toggle 右侧显示当前主题文字（"暗色"或"亮色"）
- [ ] 切换主题时所有页面颜色正确变化（首页、下载、工具、设置、登录）

### 持久化
- [ ] 切换到浅色后关闭启动器，重新打开仍为浅色
- [ ] 切换到深色后关闭启动器，重新打开仍为深色
- [ ] 首次安装（无 localStorage）默认为暗色

### 防闪烁
- [ ] 启动器启动时无白屏闪烁（暗色主题下）
- [ ] 启动器启动时无深色闪烁（浅色主题下）

### 后端同步
- [ ] 切换主题时 WebView2 背景色与主题一致（暗色=#1A1A1C, 浅色=#F7F7FA）
- [ ] 切换主题时 WPF 标题栏背景色与主题一致
- [ ] 切换主题时标题栏文字颜色与主题一致（暗色=白色, 浅色=#1D1D1F）
- [ ] 启动器启动时后端自动读取前端主题并同步背景色

### 过渡动画
- [ ] 主题切换时有 0.2s 平滑过渡动画
- [ ] 日常 hover/active 微交互不受影响
- [ ] 过渡完成后 `.theme-transitioning` 类被移除

## UI 自查

### 暗色主题（回归测试）
- [ ] 所有文字可读，对比度达标
- [ ] 卡片、边框、按钮颜色正确
- [ ] 滚动条样式正确
- [ ] Toast 通知颜色正确
- [ ] Toggle 开关颜色正确

### 浅色主题（新功能）
- [ ] 所有文字可读，对比度达标
- [ ] 卡片背景为白色/浅灰，边框可见
- [ ] 按钮颜色正确（主色按钮蓝色 + 白字）
- [ ] 滚动条样式在浅色背景下可见
- [ ] Toast 通知在浅色背景下可读
- [ ] 输入框边框和背景在浅色下正确
- [ ] 侧边栏背景和文字在浅色下正确
- [ ] 服务器状态指示器在浅色下可见
- [ ] 更新通知横幅在浅色下可读

### 响应式与适配
- [ ] Toggle 开关有 hover 效果
- [ ] Toggle 开关有 active scale(0.97) 按压反馈
- [ ] 与原设置页风格完全一致

## 代码质量

- [ ] `pnpm build` 无 TypeScript 编译错误
- [ ] `pnpm build` 无 Svelte a11y 警告
- [ ] theme.svelte.ts 导出接口清晰（current getter, init, toggle）
- [ ] IPC 命名遵循现有惯例（`theme.set`）
- [ ] localStorage key 命名遵循现有惯例（`baihe_theme`）
- [ ] 无硬编码颜色值残留（所有颜色通过 CSS 变量引用）
- [ ] TitleBar.xaml.cs 的 SetTheme 方法有异常处理

## 异常处理

- [ ] localStorage 被清除时回退到暗色默认值
- [ ] IPC theme.set 调用失败时前端主题仍可切换
- [ ] NavigationCompleted 中 ExecuteScriptAsync 失败时使用默认暗色
- [ ] TitleBar.SetTheme 被调用时 TitleText 引用不为 null
