## 1. 决策问题

如何从零自主开发一个「白鹤服务器」专属 Minecraft 启动器，使其同时满足：

1. **UI 完全按 zip 设计图还原**（macOS 风格、Tailwind CSS 4、Lucide 图标、毛玻璃侧边栏、暗色模式、Apple HIG 设计语言）
2. **云电脑 / 学校机房 / 无管理员权限环境零配置一键运行**（无需预装 .NET / Java / WebView2）
3. **完全自主开发**（不 fork PCL2-CE，代码自主重写，可参考其逻辑）
4. **预置游戏文件 + Java 21 JRE + 10 个 Fabric 模组，安装即玩**
5. **强制直连 `play.simpfun.cn:28230`，游戏窗口标题"白鹤服务器"**

## 3. 备选方案对比

### 3.1 决策矩阵

| 方案 | 设计还原度 | 核心库成熟度 | 云电脑兼容 | 体积 | 自主度 | 工作量 | 结论 |
|---|---|---|---|---|---|---|---|
| **C# .NET 10 + WebView2 + SvelteKit** ⭐ | ★★★★★ | ★★★★ | ★★★★★ | ★★★★ | ★★★★★ | ★★★ | **采纳** |
| Tauri 2.0 + Rust + 前端 | ★★★★★ | ★★ | ★★★ | ★★★★★ | ★★★★★ | ★ | 拒绝 |
| Electron + Node.js (MCLC) | ★★★★★ | ★★★★ | ★★ | ★★ | ★★★★ | ★★★★ | 拒绝 |
| C# WPF + WPF-UI | ★★ | ★★★★★ | ★★★★★ | ★★★★★ | ★★★ | ★★★★★ | 拒绝 |
| C# Avalonia | ★★ | ★★★★ | ★★★★★ | ★★★★ | ★★★★ | ★★★★ | 拒绝 |

### 3.2 拒绝理由

- **Tauri 2.0 + Rust**：Rust 生态的 MC 启动核心库不成熟。`minecraft_launcher_core` 在 lib.rs 上停滞于 2023-02 的 0.1.2 版本（[$TRAE_REF](https://lib.rs/crates/minecraft_launcher_core)），`minecraft_launcher_core_msa` 仅 2 个不稳定版本（2024-02 的 0.2.0）。需从零实现 Microsoft OAuth、Yggdrasil、版本 manifest 解析、Fabric 安装、Java 检测，工作量是 C# 方案的 3-5 倍。
- **Electron + Node.js**：Node.js 有成熟的 MCLC 库（[$TRAE_REF](https://blog.csdn.net/gitblog_01048/article/details/147614141)），但 Electron runtime 体积 ~150 MB+，内存占用高，与云电脑目标环境冲突。
- **C# WPF**：无法还原 macOS 风格交通灯按钮、毛玻璃侧边栏（`backdrop-filter: blur(30px) saturate(180%)`）、Tailwind CSS 4 设计稿、Apple HIG 阴影层级。违背用户"按设计图"硬约束。
- **C# Avalonia**：XAML 体系难以 1:1 还原 Tailwind utility class 与 CSS 变量 token 系统，毛玻璃需平台特定实现，工作量高于 WebView2 方案。

### 3.3 选定理由（C# + WebView2 + SvelteKit）

1. **设计图完美还原**：WebView2 基于 Chromium，原生支持 `backdrop-filter`、CSS 变量、Tailwind 4、Lucide mask-icon，可 1:1 复用设计稿 HTML/CSS/JS
2. **C# MC 生态成熟**：可参考 PCL2-CE 的成熟启动逻辑（BSD/MIT 许可，代码自主重写而非 fork），降低核心实现风险
3. **云电脑零依赖**：.NET 10 自包含发布 = 无需预装 .NET；WebView2 Runtime 在 Win10 1809+ 预装（Edge Chromium 自带），缺失时嵌入固定版本兜底
4. **体积可控**：启动器 ~50 MB + WebView2 兜底 ~150 MB，游戏 1.3 GB，总包 ≤ 2.2 GB
5. **用户已有经验**：project_memory 显示已用 .NET 10 + Inno Setup 打包流程，C# 方案最契合
6. **自主开发程度**：UI 完全自主（SvelteKit 重写），核心逻辑参考 PCL2-CE 但自主重写为 `Baihe.Core.dll`

### 4.2 数据流

```
用户点击「启动游戏」
   ↓
SvelteKit 前端 → postMessage({cmd: 'launch', profileId, versionId})
   ↓
Baihe.Host WebMessageReceived → 路由到 Baihe.Core.LaunchService
   ↓
LaunchService:
   1. JavaService.DetectAsync() → 返回 jre\bin\java.exe
   2. VersionService.ResolveAsync("1.21.3") → 合并 inheritsFrom
   3. ProfileService.GetProfileAsync(profileId) → Yggdrasil/离线 token
   4. ArgumentBuilder.Build(java, version, profile, server) → JVM args + classpath
   5. Process.Start(javaw.exe, args) → 游戏进程
   6. 监听 stdout/stderr → 转发日志到前端（进度反馈）
   ↓
返回 {ok: true, pid: 12345} → 前端显示「游戏已启动」
```

### 4.3 信任边界

| 边界 | 信任级别 | 说明 |
|---|---|---|
| 前端 ↔ IPC Bridge | 半信任 | 前端不可信，IPC 命令需 schema 校验（JSON Schema） |
| IPC Bridge ↔ Baihe.Core | 信任 | 同进程，直接调用 |
| Baihe.Core ↔ Mojang API | 不信任 | HTTPS + 证书校验 + SHA1 校验下载文件 |
| Baihe.Core ↔ LittleSkin API | 不信任 | HTTPS + Yggdrasil 协议校验 |
| Baihe.Core ↔ 文件系统 | 信任 | 但禁止写入 `C:\Windows`、`Program Files` |

## 8. IPC 设计

### 8.1 命令 schema（`IpcSchema.json`）

```jsonc
{
  "commands": {
    "launch.start": {
      "request": { "profileId": "string", "versionId": "string" },
      "response": { "ok": "boolean", "pid": "number?", "error": "string?" }
    },
    "profile.list": {
      "request": {},
      "response": { "profiles": [{ "id": "string", "name": "string", "type": "offline|yggdrasil|microsoft" }] }
    },
    "profile.loginOffline": {
      "request": { "username": "string" },
      "response": { "ok": "boolean", "profileId": "string?" }
    },
    "profile.loginYggdrasil": {
      "request": { "server": "string", "username": "string", "password": "string" },
      "response": { "ok": "boolean", "profileId": "string?", "error": "string?" }
    },
    "version.list": {
      "request": {},
      "response": { "versions": [{ "id": "string", "type": "string", "installed": "boolean" }] }
    },
    "java.detect": {
      "request": {},
      "response": { "path": "string", "version": "string" }
    },
    "log.subscribe": {
      "request": {},
      "response": { "streamId": "string" }
    }
  }
}
```

### 8.2 前端调用示例

```typescript
// src/lib/ipc.ts
const ipc = {
  async invoke<T>(cmd: string, args: unknown): Promise<T> {
    const id = crypto.randomUUID();
    return new Promise((resolve, reject) => {
      const handler = (e: MessageEvent) => {
        if (e.data?.id === id && e.data?.cmd === cmd) {
          window.removeEventListener('message', handler);
          e.data.ok ? resolve(e.data.response) : reject(new Error(e.data.error));
        }
      };
      window.addEventListener('message', handler);
      window.chrome.webview.postMessage({ id, cmd, args });
    });
  }
};

// 使用
const result = await ipc.invoke('launch.start', { profileId: 'p1', versionId: '1.21.3' });
```

### 8.3 C# 路由示例

```csharp
// Baihe.Host/Ipc/IpcRouter.cs
public sealed class IpcRouter {
    private readonly LaunchService _launch;
    private readonly ProfileService _profiles;

    public async Task<string> HandleAsync(string json) {
        var msg = JsonSerializer.Deserialize<IpcMessage>(json);
        return msg.Cmd switch {
            "launch.start" => await HandleLaunch(msg),
            "profile.list" => await HandleListProfiles(msg),
            "profile.loginOffline" => await HandleLoginOffline(msg),
            // ...
            _ => Error(msg.Id, "unknown_command")
        };
    }
}
```

### 9.2 QuickPlay 直连

```csharp
    private const string Server = "play.simpfun.cn";

    public IReadOnlyList<string> Build(VersionInfo version) {
        // MC 1.20+ 使用 QuickPlay，老版本用 --server --port
        if (version.Id is { } id && id.Split('.').Length >= 2
            && int.TryParse(id.Split('.')[0], out var major) && major >= 1
            && int.TryParse(id.Split('.')[1], out var minor) && (major > 1 || minor >= 20)) {
            return new[] {
                "--quickPlayMulti", $"\"{Server}:{Port}\""
            };
        }
        return new[] { "--server", Server, "--port", Port.ToString() };
    }
}
```

## 14. 责任与可逆性

- **责任**：你（单人）负责 C# 核心 + 前端 + 打包；本地检查路径为 `pnpm test:visual` + `dotnet test` + 干净 VM 验证
- **可逆性**：双向门
  - 前端框架（SvelteKit ↔ Vue ↔ SolidJS）可替换（WebView2 与框架无关）
  - C# 核心模块可独立替换（`Baihe.Core.*` 各上下文独立）
  - WebView2 可降级为 Electron（同前端代码 100% 复用，仅换宿主层）
- **改弦易辙触发器**：
  1. 若 WebView2 在 ≥3 台目标云电脑均未预装且无法静默安装 → 切换到 Tauri 2.0 + Rust（前端代码 100% 复用）
  2. 若 C# MC 核心自主实现工作量超过预估 2 倍 → 评估直接 fork PCL2-CE 核心逻辑（保留 UI 自主）
