# 白鹤服务器启动器 - 开发规格文档

> 版本: 1.0 | 最后更新: 2026-07-21
> 基础框架: PCL2 Community Edition (PCL-CE) | .NET 10 WPF
> 目标: 为白鹤服务器定制专属 Minecraft 启动器，实现零配置一键游玩

---

## 1. 项目概述

### 1.1 项目目的

为白鹤服务器（Minecraft 服务器）制作一个定制化启动器，让玩家下载安装后无需任何配置即可直接进入服务器。启动器基于 PCL2 Community Edition 开源项目二次开发，通过 UI 定制、功能裁剪、游戏文件预置、Java 运行时捆绑等方式，实现"安装即玩"的体验。

### 1.2 服务器信息

| 项目 | 值 |
|------|-----|
| 服务器名称 | 白鹤服务器 |
| 服务器地址 | `play.simpfun.cn:28230` |
| 游戏版本 | Minecraft 1.21.3 |
| 模组加载器 | Fabric Loader 0.16.14 |
| Java 版本 | Java 21 (JRE) |

### 1.3 目标用户环境

- Windows 10 1809+ 64 位
- 可能没有管理员权限（如云电脑、学校机房）
- 可能没有预装 Java 或 .NET 运行时
- 用户名可能非英文（如云电脑的 `remote` 用户）
- 需要跨用户环境一致性

### 4.3 启动器自动检测机制

#### 4.3.1 游戏文件夹检测

启动器在 `ModFolder.cs` 的 `McFolderListLoadSub()` 中按以下顺序扫描:

1. 读取自定义文件夹配置（首次为空）
2. 检查 `exePath\versions\` 是否存在
3. **遍历 `exePath` 下所有子目录**，如果子目录名 == `.minecraft` 或包含 `versions` 子目录，添加到列表
4. 检查 `%AppData%\.minecraft\versions\` 是否存在（官启文件夹）
5. 若列表为空，自动创建 `exePath\.minecraft\versions\`

**关键**: 第 3 步确保 `Qihang.exe` 同级的 `.minecraft` 文件夹会被自动检测到。

#### 4.3.2 Java 检测

启动器在 `ModJava.cs` 中按以下优先级查找 Java:

1. 全局配置指定的 Java
2. **捆绑 Java**: `exePath\jre\bin\java.exe`（启航定制新增）
3. 自动搜索系统中的 Java

**关键**: 第 2 步确保捆绑的 JRE 会被优先使用。

#### 4.3.3 选中文件夹的 `$` 占位符机制

配置中使用 `$` 作为 `exePath` 的占位符:
- 写入: `folder.Location.Replace(exePath, "$")` → `$.minecraft\`
- 读取: `SelectedFolder.Replace("$", exePath)` → `exePath\.minecraft\`

这使得配置在跨机器迁移时不需要修改路径。

### 6.2 Inno Setup 脚本配置

#### 6.2.1 应用信息

```inisetup
#define MyAppName "白鹤服务器启动器"
#define MyAppVersion "1.0.0"
#define MyAppPublisher "白鹤"
```

#### 6.2.2 安装目录与权限

```inisetup
; 安装到用户目录，避免 Program Files 写入权限问题
DefaultDirName={localappdata}\白鹤服务器
PrivilegesRequired=lowest
ArchitecturesAllowed=x64
ArchitecturesInstallIn64BitMode=x64
```

**关键**: 必须安装到 `{localappdata}` 而非 `{autopf}`（Program Files）。原因:
- 云电脑/学校机房可能无管理员权限
- Program Files 目录写入会导致日志文件创建失败
- `PrivilegesRequired=lowest` 确保无需 UAC 提权

#### 6.2.3 界面配置

```inisetup
WizardStyle=modern
DisableWelcomePage=no
DisableReadyPage=no
ShowLanguageDialog=no
LanguageDetectionMethod=none
```

**关键**: 不使用 IsSkin.dll。IsSkin.dll 会在提取失败时导致 Access Violation 崩溃。使用 Inno Setup 原生 modern 界面即可。

## 7. 踩过的坑

### 7.1 .NET 单文件发布崩溃

**问题**: `PublishSingleFile=true` 时，Qihang.exe 在 Program Files 或其他受限目录中运行，报错 `0x80008009`（Failed to create default extraction directory, Access denied）。

**原因**: .NET 单文件应用在运行时需要将 bundled DLL 解压到临时目录。在受限目录中，解压失败导致崩溃。

**解决**: 设置 `PublishSingleFile=false`，保留独立的 DLL 文件。虽然文件数量增多，但稳定性大幅提升。

### 7.2 Program Files 写入权限

**问题**: 安装到 `C:\Program Files\白鹤服务器\` 后，启动器无法在自身目录创建 `PCL\Log\` 目录，导致写入失败。

**解决**: 将安装目录改为 `{localappdata}\白鹤服务器`（即 `C:\Users\用户名\AppData\Local\白鹤服务器`），`PrivilegesRequired=lowest`。

### 7.3 IsSkin.dll 崩溃

**问题**: Inno Setup 脚本通过 `LoadSkin@files:IsSkin.dll stdcall` 加载 Vista 皮肤 DLL，安装时触发 Access Violation 崩溃。

**原因**: IsSkin.dll 在提取过程中失败，但 Inno Setup 仍尝试调用其中的函数。

**解决**: 完全移除 IsSkin.dll 和所有外部 DLL 加载代码，使用 Inno Setup 原生 modern 界面。

### 7.4 中文翻译不完整

**问题**: 使用 `ChineseSimplified.isl` 作为语言文件，安装界面中英混杂。

**原因**: `ChineseSimplified.isl` 是社区贡献的残缺翻译，大量字符串未翻译。

**解决**: 使用 `English.isl` 作为基底，通过 `[CustomMessages]` 段全量覆盖为中文。所有按钮、页面标题、提示文本均需手动翻译。

### 7.5 图标格式错误

**问题**: `SetupIconFile` 指向 `.bmp` 文件，Inno Setup 编译报错。

**解决**: 使用 `.ico` 格式图标（256x256），存放在 `installer_resources\icon.ico`。

### 7.6 中文文件名文件锁

**问题**: `OutputBaseFilename` 使用中文"白鹤服务器启动器_安装包_v1.0"，编译时出现"另一个程序正在使用此文件"错误。

**解决**: `OutputBaseFilename` 改为英文 `BaiheServer_Setup_v1.0`。

### 7.7 .minecraft 放置位置错误

**问题**: 游戏文件安装到 `{localappdata}\.minecraft` 或 `{app}\.minecraft`，启动器无法检测到游戏。

**原因**: 启动器在 `ModFolder.cs` 中扫描 `exePath` 下的子目录，`.minecraft` 必须是 `Qihang.exe` 的同级子目录。

**解决**: 将 `.minecraft` 的 `DestDir` 设为 `{app}\启动器\.minecraft`，确保与 `Qihang.exe` 在同一目录下。

### 7.8 模组文件名 emoji 问题

**问题**: Advanced Installer 的 .aip 文件是 XML 格式，模组文件名中的 emoji（如 `🔍`）导致 XML 解析错误。

**解决**: 放弃 Advanced Installer，改用 Inno Setup。或重命名含 emoji 的文件。

### 7.9 开发日志泄漏路径

**问题**: `PCL\Log\` 目录下的日志文件包含开发机器的绝对路径和用户名（如 `C:\Users\admin.DESKTOP-4R9CSJI\`），打包进安装包会泄漏信息。

**解决**: 在 Inno Setup 脚本的 `Excludes` 中添加 `PCL\Log\*` 和 `PCL\*.bak`。
